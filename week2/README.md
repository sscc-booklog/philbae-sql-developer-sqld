제1정규형 - 원자성 보장(컬럼에 여러 값 X, 인스턴스 내에서 같은 속성 반복 X)

제2정규형 - 제1정규형 + 기본키 부분 함수 종속 제거

→ 즉, 기본키가 두개 이상이고 어떤 속성들이 각각의 기본키에만 종속될 때 따로 분리해서 테이블을 만드는 것

안했을 때 문제점: 중복이 발생할 수 있음, 기본키들 중 어떤 기본키에만 종속된다면 종속되지 않은 다른 속성 내에서는 속성값이 중복이 발생할 수 있는데 그러면 갱신 이상이 발생함. 따라서 테이블을 따로 빼서 한번만 갱신할 수 있도록 해야함.

제3정규형 - 제2정규형 + 이행 함수 종속 제거

→ 이중적으로 이어진다면 분리하기(원래 주요 키랑은 관련없으니까 이중적으로 종속적인 속성들은 분리해서 테이블을 만드는 것), 일반 속성끼리 종속X

제2정규화는 기본키 2쌍 이상일 때, 제3정규화는 1쌍일 때 이행 함수 있는 걸로 구분하기, 제1정규화는 속성이 여러개 있는 경우

정규화: 중복 감소(제2정규형), 관심사 묶임(제3정규형), 조인이 발생해서 성능 저하 단점

반정규화: 조인을 해서 성능을 향상시키는 것

컬럼 반정규화(컬럼 수 늘리는거): 중복 컬럼 추가, 파생 컬럼 추가(aggregate function), 이력 테이블 컬럼 추가, 의미적 분리 컬럼 추가, 데이터 복구 컬럼 추가

테이블 반정규화(테이블 합치는거): 테이블 병합(관계, 슈퍼/서브), 테이블 분할(수직, 수평-파티셔닝), 테이블 추가(중복 추가, 통계(aggregate) 추가, 이력 추가, 부분 추가(원본 유지하면서 부분 테이블 생성)

인덱스: 검색 속도 향상 but 저장 공간 및 갱신, 삽입, 삭제 시간 늘**어남**
